# 批改网自动写作工作流程

## 概述
通过 Playwright MCP 工具远程操控批改网（pigai.org），实现自动登录、搜索作业、撰写作文、提交批改、迭代优化。

## 完整流程

### 1. 打开网站
```
访问 https://www.pigai.org/
```

### 2. 自动登录
- 用户提供账号和密码后，自动填入并登录
- **手机号输入框**：`input[placeholder*="手机号"]`
- **密码输入框**：`input[placeholder="密码"]`
- 填写后按 Enter 键提交登录（页面无显式登录按钮）

### 3. 搜索作业号
- 搜索框定位：`input[name="rid"]`
- 填入作业号后点击按钮：`button:has-text("查 找 作 文")`

### 4. 查看题目要求
页面会显示：
- 作文标题与题目描述
- 教师姓名
- 字数要求
- 截止时间

### 5. 填写作文
- **标题输入框**：`input[name="title"]`
- **正文区域**：`textarea[name="contents"]`（class: `from_contents`）
- 使用 `browser_type` 的 fill 方法填入内容

### 6. 选择班级
- 班级下拉框：`select:has(option:has-text("班级名"))`  


### 7. 提交作文
- 提交链接定位：查找包含文本"提交作文"的 `<a>` 标签并点击
- 如果出现弹窗"请确认勿选错班级"，先关闭再提交

### 8. 查看评分
提交后页面自动跳转到评分结果页，主要关注：
- **总分**（页面中部大号数字）
- **词汇分数**（Vocabulary）
- **句子分数**（Sentence）
- **篇章结构**（Structure）
- **内容相关**（Coherence）
- **评语**（Teacher comment） 注意：部分评语即使在做出相应修改之后依旧出现甚至自相矛盾，注意甄别

### 9. 迭代优化策略
若无分数特别要求则：
| 分数范围 | 处理方式 |
|---------|---------|
| < 91 分 | 按评分建议重写文章，重新提交 |
| 91 ≤ 分数 < 94 | 按评分建议逐项修改最多三次后重新提交，超过三次则启动重写 |
| ≥ 94 分 | 按建议微调后提交，结束流程 |

若有特殊分数要求
| 分数范围 | 处理方式 |
|---------|---------|
| （要求-分数）＞ 3 分 | 按评分建议重写文章，重新提交 |
| 0 ＜ （要求-分数） ≤ 3分 | 按评分建议逐项修改最多三次后重新提交，超过三次则启动重写 |
| （要求-分数）＜0 分 | 用更低级的逻辑、语法、词汇、结构等重写文章以降低分数并重新提交 |
| （要求-分数） ＝ 0 | 提交，结束流程 |


### 10. 回到编辑页面
URL 格式：
```
https://www.pigai.org/index.php?c=v2&a=write&eid={eid}&rid={作业号}
```
或点击 "还可以继续修改完善" 链接。

## 常见问题及修复

### 字数显示为 0
- 原因：通过 JS 填入内容未触发页面的 input 事件
- 修复：对 textarea 手动 dispatch input 事件
```javascript
ta.focus();
ta.dispatchEvent(new Event('input', { bubbles: true }));
ta.blur();
```

### 弹窗处理
选择班级后弹出的提示框，点击"关闭"按钮关闭。
定位方式：查找包含文本"关闭"的可点击元素。

### CSS 选择器参考

| 账号输入框 | `input[placeholder*="手机号"]` |
| 密码输入框 | `input[placeholder="密码"]` |
| 作业搜索框 | `input[name="rid"]` |
| 搜索按钮 | `button:has-text("查 找 作 文")` |
| 标题输入框 | `input[name="title"]` |
| 正文区域 | `textarea[name="contents"]` |
| 班级下拉框 | `select:has(option:has-text("班级名"))` |

## 作文评分维度
1. **词汇（Vocabulary）**：词汇丰富度、拼写正确性
2. **句子（Sentence）**：句法复杂度、从句使用、长短句结合
3. **篇章结构（Structure）**：段落组织、文章结构
4. **内容相关（Content Relevance）**：词句段之间的内容连贯性、逻辑严谨

## 注意事项
- 每次修改后保留未错误内容，只针对性地修正系统标记的问题
